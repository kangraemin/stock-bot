# Analysis Output Rules

백테스트/분석 결과를 사용자에게 제시할 때 반드시 지켜야 하는 규칙.

## 필수 포함 항목

분석 결과 테이블이나 비교표를 출력할 때, **모든 행**에 다음 두 항목을 반드시 포함:

1. **거래 횟수** (trades) — 전략이 몇 번 매매했는지
2. **1년 평균 거래 횟수** (trades/yr) — 총 거래 횟수 / 분석 기간(년)
3. **vs B&H** — Buy & Hold 대비 초과/미달 수익률 (%p)
4. **분석 기간** — 몇 년간 데이터인지 (예: 5y, 2020~2025, 1,260일)
5. **타임프레임** — 일봉(daily) / 시간봉(1h) / 주봉(weekly) 중 어떤 데이터 기준인지

이 다섯 항목이 없는 분석 결과는 불완전하다. 절대 생략하지 않는다.

## 비교 기준선

- **SHV** (iShares Short Treasury Bond ETF, 단기 국채) 를 무위험 수익률 기준선으로 사용
- SHV 실측: CAGR 1.52%, MDD -0.45% (2007~2026, 19년)
- 포트폴리오/전략 분석 시 SHV 대비 초과수익과 추가 리스크를 함께 제시

## 최소 거래 횟수 기준

- **1년 평균 거래 횟수 < 5회**인 전략은 **사용 불가** 판정한다.
- 결과 테이블에 포함하되, "거래 부족" 경고를 표시하고 추천하지 않는다.
- 거래 1~2회로 높은 수익률이 나와도 통계적 우연일 뿐, 전략으로 채택하지 않는다.

## 이유

- 거래 횟수가 적으면 (< 10) 통계적으로 무의미한 결과
- vs B&H 없이 수익률만 보면 "그냥 들고 있는 게 나은데?" 판단 불가
- Sharpe만으로는 실전 가치를 판단할 수 없음
- SHV 기준 없이 수익률만 보면 "그냥 단기채 사는 게 안전한데?" 판단 불가
